  数据库，我来给你建议单词数据库应该保存的字段：

  单词数据库设计建议

  核心表：words

  | 字段              | 类型     | 说明                            | 来源
            |
  |-----------------|--------|-------------------------------|--------------
  --|
  | word            | string | 单词原形                          | -
        |
  | pos             | string | 词性 (noun/verb/adj/adv/phrase) | cefrpy /
  spaCy |
  | cefr_level      | enum   | A1/A2/B1/B2/C1/C2             | cefrpy
    |
  | frequency_rank  | int    | 词频排名 (COCA/BNC)               |
  外部词频表          |
  | is_cet4         | bool   | 是否 CET4 词汇                    | 词表匹配
            |
  | is_cet6         | bool   | 是否 CET6 词汇                    | 词表匹配
            |
  | is_common_3000  | bool   | 是否常用 3000 词                   | 词表匹配
             |
  | transferability | float  | 可迁移性分数 (多场景复用度)               |
  语料统计           |

  短语/搭配表：phrases

  | 字段              | 类型     | 说明
               |
  |-----------------|--------|----------------------------------------------
  -------|
  | phrase          | string | 短语原文
             |
  | type            | enum   | phrasal_verb / collocation / idiom /
  clause_starter |
  | pmi_score       | float  | 共现概率分数
               |
  | translation_gap | float  | 翻译不对称性分数
                 |
  | register        | enum[] | spoken / formal / academic / business
         |
  | cefr_level      | enum   | 短语整体难度
               |

  用户学习记录表：user_word_status

  | 字段              | 类型          | 说明                            |
  |-----------------|-------------|-------------------------------|
  | user_id         | string      | 用户 ID                         |
  | word_id         | foreign key | 关联单词                          |
  | status          | enum        | unknown / learning / mastered |
  | encounter_count | int         | 遇到次数                          |
  | last_seen_at    | timestamp   | 最后遇见时间                        |
  | feedback        | enum        | useful / too_easy / mastered  |

  为什么这样设计？

  1. 过滤层：is_cet4/6 + is_common_3000 + cefr_level
  可以快速过滤掉不值得学的词
  2. 价值层：frequency_rank + transferability 判断语言价值
  3. 个性化：user_word_status 实现"同词不同人价值不同"
  4. 短语独立：短语单独建表，因为评分逻辑完全不同
