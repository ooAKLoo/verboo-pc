# 目标人群

- 考研用户：侧重学术词汇 + 逻辑连接词
- 职场用户：侧重商务表达 + 会议常用短语
- 影视用户：侧重口语化表达 + 文化梗

## 一、单词：什么是"重点词汇"

| 维度 | 定义 | 筛选规则 | 例子 |
|------|------|----------|------|
| **语言层 Language Value** | 词本身是否值得学 | • 不在 CET4/6/常用3000词内<br>• 词性优先级：动词短语 > 抽象名词 > 形容词<br>• 可迁移性：能在多场景复用 | `take a toll on` 比 `happy` 更值得学 |
| **场景层 Context Value** | 在这段视频里为什么重要 | • 是否承载观点<br>• 是否用于表达立场/转折/强调 | `I'd argue that...`、`What really matters is...` |
| **表达层 Expression Value** | 学了能不能更像 native | • 是否非直译中文<br>• 是否是语块/固定搭配 | `make sense` ≠ 制造意义 |
| **用户层 User Value** | 对这个用户是否重要 | • 是否已认识<br>• 学过但没掌握<br>• 是否在学习目标中 | 同一个词对不同用户价值不同 |

**过滤规则（排除）：**
- CET4/6 常用词
- 功能词（冠词、代词、助动词）
- 用户已掌握的词

## 二、短语：什么是"地道表达"

| 识别路径 | 核心逻辑 | 具体规则 | 例子 |
|----------|----------|----------|------|
| **统计学：n-gram + PMI** | 词组共现概率远高于随机 | 计算 bi-gram/tri-gram 的 PMI、t-score | ✅ `take into account`<br>❌ `take the book` |
| **句法结构过滤** | 符合高价值句法模式 | 保留：Verb+Prep、Verb+Obj、Prep+NP、Clause starter<br>过滤：冠词+名词、代词+助动词 | ✅ `rely on`、`in the long run`<br>❌ `the book`、`I will` |
| **翻译不对称性** | 整体翻译 ≠ 逐词翻译之和 | 逐词翻译 vs 整体翻译差异大 → 是 idiom | `take a toll on`<br>❌ 拿一个收费 → ✅ 造成损耗 |
| **Register 信号** | 口语/访谈场景高频 | 是否在 spoken/interview/informal 语料中高频出现 | `What's interesting is...` |

**短语评分公式：**
```
Expression Score = 
  Collocation Score (PMI) 
+ Structural Score (句法模式)
+ Translation Gap (翻译差异)
+ Register Signal (场景匹配)
+ LLM Judgment (20%权重)
```

## 三、LLM 模块设计

**核心原则：AI 不是用来"找"，而是用来"判"**

| 阶段 | 输入 | LLM 任务 | 输出 |
|------|------|----------|------|
| **单词判定** | 机器筛选的候选词（已过滤常用词） | 判断：这个词在当前语境中承载什么意义？ | 重要性标签 + 理由 |
| **短语判定** | 统计+句法筛选的候选短语（几十个） | 判断：是否常用表达？native 会不会这么说？适合学习吗？ | 是否保留 + 标签 |
| **理由生成** | 最终选定的词/短语 | 生成用户可读的学习理由 | `高频表达 · 面试常用` |

**LLM Prompt 设计要点：**

```markdown
## 输入
- 候选词/短语列表（已经过机器预筛选）
- 原句上下文
- 用户画像（可选）

## 任务
1. 判断每个候选是否值得学习（Yes/No）
2. 如果 Yes，给出理由标签：
   - 高频表达 / 访谈常用 / 表达观点关键词 / 非中式英语 / native 常用说法
3. 给出适用场景：面试 / 日常 / 工作 / 学术

## 输出格式
{
  "phrase": "take a toll on",
  "keep": true,
  "tags": ["高频表达", "非中式英语"],
  "scene": ["面试", "工作"],
  "reason": "表示长期负面影响，比 affect 更地道"
}
```
